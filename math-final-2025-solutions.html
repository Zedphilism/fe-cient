QUESTION 1 — ACCURACY & NON-LINEAR EQUATIONS
QUESTION 1 [20 MARKS]

a) Given f(x) = 4x³ − 2x² + 1: (i) find the exact value of f(2.13). (ii) find the absolute and relative error if f(2.13) is rounded to 3 decimal points.

b) Find the root of f(x) = x³ − 4x + 1 to 1 decimal point accuracy in [1, 3] using the Bisection method. Calculate to 3 decimal points.

c) Find the root to 2 decimal points for f(x) = cos(x) − x using the Secant method, where x₀ = 1 and x₁ = 1.5. Calculate to 4 decimal points.

🧠 FUNDAMENTAL CONCEPT

(a) is really asking: how much do we lose when we round? Every number a computer or calculator stores is truncated somewhere — error analysis is just the honest bookkeeping of that loss, split into a raw amount (absolute) and a size-adjusted amount (relative).

(b, c) are both really the SAME problem — "where does this curve cross zero?" — solved two different ways. Bisection treats it as a search problem: repeatedly ask "is the root in the left half or the right half?" (this only works because a continuous function can't skip over zero — the Intermediate Value Theorem). Secant treats it as a local-slope problem: draw a straight line through your two best guesses and jump to where THAT line crosses zero, using the slope between two points as a stand-in for the derivative.

🎯 HOW TO APPROACH IT

(a) Straight substitution, then apply the absolute/relative error formulas from Chapter 5 — no iteration needed.

(b) Verify IVT first (always), then bisect using x_c=(a+b)/2, tracking a, b, f(a), f(c) every round. Stop once the digit at the target decimal place stabilises across consecutive iterates.

(c) Secant needs no sign-change bracket — just two starting points and the formula from the sheet. Always slide the window forward (drop the oldest point) each round.

QUESTION 2 — EIGENVALUES & EIGENVECTORS
QUESTION 2 [20 MARKS]

a) Given A = [[4,1],[2,3]]: (i) use the characteristic polynomial to find the eigenvalues and eigenvectors. (ii) show Σλᵢ = Σaᵢᵢ (trace).

b) Given B = [[4,1,−1],[2,3,1],[−1,1,2]]: (i) find the smallest-magnitude eigenvalue λ₃ and its eigenvector using the shifted power method, given the largest eigenvalue λ₁ = 5, starting v₀ = [0.5, −0.4, 0.5]ᵀ, iterate until ε = 0.05. (ii) find the intermediate eigenvalue λ₂.

🧠 FUNDAMENTAL CONCEPT

Most directions get twisted AND stretched when a matrix acts on them. An eigenvector is a special direction that only gets stretched (never rotated) — the eigenvalue is simply how much. The characteristic polynomial is the algebraic trick for finding these directions exactly: it asks "for which λ does (A−λI) collapse a whole direction to zero?" (i.e. becomes singular).

The power method exploits a different idea entirely — repeated multiplication by A stretches the DOMINANT eigenvector's direction exponentially faster than every other direction, so ANY starting vector eventually "forgets" every direction except the dominant one. Shifting first (subtracting λ₁I) temporarily makes a DIFFERENT eigenvalue the dominant one, letting the same trick find the smallest instead of the largest.

🎯 HOW TO APPROACH IT

(a) For a 2×2 matrix, use the fast shortcut λ² − (trace)λ + det = 0 instead of expanding the full determinant — much less room for error.

(b)(i) "Shifted power method" here means: form A_shifted = B − λ₁I (subtract the GIVEN largest eigenvalue from the diagonal), then run the ORDINARY power method on A_shifted. Its dominant eigenvalue m corresponds to (λ₃ − λ₁), so recover λ₃ = m + λ₁ at the end.

(b)(ii) Don't re-run another iteration — use Σλᵢ = trace(B): λ₂ = trace(B) − λ₁ − λ₃.

For 2×2 matrices, NEVER expand the full determinant by hand — always jump straight to λ²−(trace)λ+det=0. For ANY size matrix, sum of eigenvalues = trace and product of eigenvalues = det — use these as free checks (or shortcuts) whenever you're missing just one eigenvalue.

💡 REMEMBER

"Shifted power method for the smallest eigenvalue" = power method on (A − λ_largest·I), then add λ_largest back at the end. Never solve for the middle eigenvalue by iterating — use Σλᵢ = trace instead, it's instant.

QUESTION 3 — INTERPOLATION & LEAST SQUARES
QUESTION 3 [20 MARKS]

An object's position is recorded: t = 1,2,3,4,5 → position = 2.0, 3.5, 7.8, 13.9, 21.0 (metres).

a) Estimate the position at t = 4.4 using (i) the appropriate Newton method, and (ii) the Lagrange interpolating polynomial (L₀=−0.03, L₁=−0.19, L₂=−0.49 given).

b) Find p(x) = a₀ + a₁x using Least Squares Approximation. (All calculations to 2 d.p.)

🧠 FUNDAMENTAL CONCEPT

Give a mathematician n+1 points, and there is exactly ONE polynomial of degree ≤n that threads through all of them exactly — interpolation is the art of finding that one polynomial without solving a huge system of equations directly. Newton's difference formulas and Lagrange's formula are two completely different arithmetic ROUTES to the SAME destination (which is exactly why they agreed at 16.66 above).

Least squares is solving a DIFFERENT problem: when your data doesn't lie on a clean polynomial at all (it's noisy/measured), don't force a curve through every point — instead find the line that minimises the TOTAL squared vertical distance to all points. It's a best compromise, not an exact fit.

🎯 HOW TO APPROACH IT

(a)(i) t=4.4 sits near the END of the table (closest to t=5) → backward difference is the "appropriate" Newton method here (forward would need many terms to converge from the far end).

(a)(ii) With 5 data points, Lagrange uses ALL FIVE (degree 4) — the 3 L-values given are a hint/check, not the full set; you must derive L₃ and L₄ yourself. Sanity check: all Lᵢ(x) must always sum to exactly 1 — use this to catch arithmetic slips.

(b) Standard linear least-squares normal equations using all 5 (t, position) pairs.

💡 REMEMBER

All Lagrange basis values at ANY x must sum to exactly 1.0 — use this to check your arithmetic before submitting. Near table END → backward difference; near table START → forward.

QUESTION 4 — NUMERICAL DIFFERENTIATION
QUESTION 4 [20 MARKS]

Table 2 gives f(x) = ln(x²+1) at x = 0.9 to 1.4 (step 0.1): f = 0.58779, 0.69315, 0.78846, 0.87547, 0.95551, 1.02962.

a) Estimate f′(1.0) with h=0.1 using (i) two-point forward, (ii) three-point forward, (iii) five-point forward.

b) Given f′(x) = 2x/(x²+1) exactly, find the absolute error for each estimate in (a) and explain what more data points do to accuracy.

c) Estimate f″(1.1) using the five-point central difference formula, h=0.1.

🧠 FUNDAMENTAL CONCEPT

A derivative is technically a LIMIT as h→0 — but with only tabulated data, you can never actually take h to zero. Numerical differentiation approximates that limit using a small but finite h, by writing a Taylor series around the point and solving for f′(x) in terms of nearby f-values. Using MORE nearby points lets you cancel out more of the leftover (truncation) terms, which is exactly why 2-point → 3-point → 5-point gets progressively more accurate — you are literally cancelling more of the Taylor-series error with each extra point.

🎯 HOW TO APPROACH IT

All the data you need (x=1.0 to 1.4 for part a/b, x=0.9 to 1.3 for part c) already exists in Table 2 — no extra values need computing here. Just plug straight into each named formula from the formula sheet.

(b) Compare each estimate against the exact f′(1.0)=2(1)/(1+1)=1.0 — expect the error to shrink noticeably as you use more points (higher-order formula).

🔑 KEY TRICK

When a question gives you both the table AND the analytical derivative, always compute BOTH and compare — it's a free accuracy sanity check and usually IS the next sub-question. Higher-order formulas have large coefficients (±25, ±48…) that amplify tiny data-rounding errors, so don't be alarmed if a high-order estimate looks "too different" from a simple analytical check on imprecise data — the method is still correct, the input data is just imperfect.

💡 REMEMBER

Central difference is ALWAYS preferred when both neighbours exist — it's one order more accurate than the one-sided version at the same point count. Forward-only formulas exist for boundary points where you can't look backward.

QUESTION 5 — NUMERICAL INTEGRATION
QUESTION 5 [20 MARKS]

Approximate ∫₁⁴ [(x²−x+1)/x³] dx using: a) Trapezoidal, N=6. b) Simpson's 1/3, N=6. c) Romberg integration, calculate until R₃,₃.

🧠 FUNDAMENTAL CONCEPT

An integral is the exact area under a curve — but for most functions you cannot find that area with a clean formula. So instead, replace the curve with simpler SHAPES whose area you CAN compute exactly, and add those up. Trapezoidal replaces each slice with a straight-line (linear) top; Simpson's replaces pairs of slices with a parabola (quadratic) — a better-fitting shape, hence more accuracy for free. Romberg is cleverer still: it runs the crude trapezoidal rule at a few different step sizes, then mathematically CANCELS OUT the leading error term between them (Richardson extrapolation) — squeezing a much better estimate out of the same "unsophisticated" trapezoidal building block.

🎯 HOW TO APPROACH IT

Same integral, three methods — compute the node values ONCE (h=(4−1)/6=0.5, 7 nodes) and reuse them for (a) and (b). For (c), Romberg needs its OWN doubling sequence starting from N=1 (h₁=3), NOT the N=6 nodes from parts (a)/(b) — build R₁,₁ → R₂,₁ → R₃,₁ first (trapezoidal at N=1,2,4), then Richardson-extrapolate across to R₃,₃.

🔑 KEY TRICK

Simpson's needs N EVEN — if you're ever handed an odd N, split off the last panel and handle that one with trapezoidal (or the last 3 panels with Simpson's 3/8), then apply 1/3 to the rest. When building a Romberg table, use R_{i,2} to sanity-check against a direct Simpson's calculation at that same step size before extrapolating further.

💡 REMEMBER

Romberg's R_{i,2} column always equals Simpson's 1/3 at that same refinement level — a great self-check. R₃,₂ = (4R₃,₁−R₂,₁)/3 uses 4¹; R₃,₃ = (16R₃,₂−R₂,₂)/15 uses 4²=16 — the power always matches the column index j via 4^(j−1).
