What is the Internet?

Nuts-and-bolts view · Service view · End systems · ISPs

EXPLAIN

The Internet is a global network of billions of interconnected devices — computers, phones, servers, sensors — exchanging data using shared protocols. It can be viewed from two angles: the nuts-and-bolts view (physical hardware and links) and the service view (an infrastructure that delivers services to applications).

  • Nuts-and-bolts: end systems (hosts) connected by communication links and packet switches
  • Communication links: fiber optics, copper wire, radio, satellite
  • Packet switches (routers + link-layer switches) forward packets toward destinations
  • End systems connect to the Internet via Internet Service Providers (ISPs)
  • Service view: the Internet is a platform delivering services to distributed applications
  • Distributed apps run on end systems — NOT on packet switches in the core
  • Protocols define the format and order of messages between communicating entities
INTERACTIVE SIMULATION
THINK OF IT LIKE…

The Internet is like a postal system. Your home is an end system, roads are communication links, sorting facilities are packet switches, and the postal service is your ISP. Your letter (data) gets broken into parcels (packets) that travel independently and are reassembled at the destination.

Protocols

Definition · Role · TCP/IP · IETF RFCs · Examples

Complete Section 1 to unlock

EXPLAIN

A protocol defines the rules governing communication between entities — the format and order of messages exchanged, and the actions taken. All activity on the Internet is governed by protocols. Without agreed-upon protocols, devices from different vendors could not communicate.

  • A protocol specifies: message format, message order, and actions on send/receive
  • Protocols exist at every layer: application, transport, network, link, physical
  • Key examples: HTTP (web), SMTP (email), TCP (reliable transfer), IP (routing), 802.11 (WiFi)
  • TCP/IP is the foundational protocol suite of the Internet
  • The IETF standardises Internet protocols via RFC documents
  • Protocols enable interoperability across different vendors and platforms
INTERACTIVE SIMULATION
THINK OF IT LIKE…

Protocols are like the rules of a formal meeting. Before speaking you raise your hand (SYN). The chairperson acknowledges (SYN-ACK). You confirm and begin (ACK). Everyone follows the same rules. Imagine if some people spoke French and others Morse code with no agreed translator — that's a network without protocols.

Network Structure

Network edge · Access network · Network core · Packet vs circuit switching

Complete Section 2 to unlock

EXPLAIN

The Internet's structure has three key parts: the network edge (end systems and applications), the access network (the last-mile link to the first router), and the network core (the mesh of high-speed routers forwarding packets globally).

  • Network edge: end systems (hosts) — clients and servers — where applications live
  • Servers often housed in large data centers (Google, AWS, Meta)
  • Access network: physical link from end system to its first (edge) router — the "last mile"
  • Network core: mesh of high-speed routers interconnected by high-bandwidth fiber links
  • Packet switching: store-and-forward; no dedicated path; used in the Internet core
  • Circuit switching (phone networks): reserves a dedicated path for the full call duration
  • Statistical multiplexing: many users share links efficiently — more efficient for bursty traffic
INTERACTIVE SIMULATION
THINK OF IT LIKE…

Picture a city. Your home is the edge. The post office entrance is the access network — the path from your door to the network. The highway system is the core. Your letter (packet) doesn't reserve a lane on the highway — it travels whenever space is available (statistical multiplexing).

Access Networks

DSL · Cable (HFC) · FTTH · Wireless · Enterprise Ethernet

Complete Section 3 to unlock

EXPLAIN

Access networks are the last-mile technologies connecting homes, businesses, and mobile devices to the first router. They vary significantly in speed, medium, and topology. Core types: DSL, cable (HFC), fiber-to-the-home (FTTH), wireless (WiFi/cellular), and enterprise Ethernet.

  • DSL — uses telephone copper wire; asymmetric (download faster than upload); 1–100 Mbps
  • DSL uses frequency division multiplexing — voice and data share the same line
  • Cable (HFC) — shared coaxial TV medium; up to 1.2 Gbps down; all neighbours share bandwidth
  • FTTH — fiber optic all the way to premises; gigabit speeds; lowest latency
  • FTTH architectures: AON (Active Optical Network) and PON (Passive Optical Network)
  • WiFi (802.11) — shared wireless access point; typically 10–100 Mbps range
  • 4G/5G cellular — wide-area wireless; 4G ~60 Mbps, 5G potentially >1 Gbps

INTERACTIVE SIMULATION
THINK OF IT LIKE…

Access networks are different roads to the same highway. DSL is a narrow country road repurposed from telephone lines. Cable is a shared suburban lane — faster, but you share with neighbours. Fiber is a private express lane with no congestion. Cellular is a helicopter — wireless and flexible, but subject to interference.
