PREDICTION 1 — CH.6 NON-LINEAR EQUATIONS
PRACTICE QUESTION 1 — predicted: Newton-Raphson untested in 2025

Find the root of f(x) = x³ − 2x − 5 to 3 decimal points using the Newton-Raphson method, starting from x₀ = 2. Show all iterations.

🧠 FUNDAMENTAL CONCEPT

At any point on a curve, the tangent line is the BEST local straight-line approximation of that curve. Newton-Raphson's whole idea: if you're standing near a root, don't search blindly — follow the tangent line down to where IT crosses zero, and that's usually a much better guess than where you started. Repeat, and the tangent gets more and more accurate as you approach the true root — which is why it converges so much faster than bisection (each step roughly DOUBLES your correct digits).

🎯 HOW TO APPROACH IT

Differentiate f(x) once, up front. Then it's pure repetition: plug xₙ into both f and f′, apply the formula, check whether consecutive x-values have stopped changing (to the required decimal places), and stop.

🔑 KEY TRICK

Before you start, glance at f′(x₀) — if it's close to zero, the tangent is nearly flat and the method can shoot off wildly. If f(x₀) and f′(x₀) are both reasonable-sized numbers (like here: −1 and 10), you're safe. Also: Newton-Raphson needs NO sign-change bracket, unlike bisection — only one starting point.

💡 REMEMBER

Newton-Raphson roughly DOUBLES correct digits every iteration (quadratic convergence) — that's why it converged here in only 3 steps versus bisection's 8. The trade-off: it needs f′(x) and a decent x₀, with no convergence guarantee.

PREDICTION 2 — CH.7 EIGENVALUES
PRACTICE QUESTION 2 — predicted: plain (unshifted) power method

Given matrix A = [[4,1,1],[1,3,0],[1,0,2]], find the dominant eigenvalue and its eigenvector using the power method. Start with v₀ = [1, 0, 0]ᵀ and iterate until ε = 0.05.

🧠 FUNDAMENTAL CONCEPT

Any starting vector is secretly a MIX of all the matrix's eigenvector directions blended together. Multiplying by A stretches each direction by its own eigenvalue — and the direction with the BIGGEST eigenvalue gets stretched the most, every single time you multiply. Do it enough times, and that dominant direction completely swamps all the others — the mixture effectively "forgets" every direction except the strongest one. That survivor IS the dominant eigenvector, and the amount it grew by each step IS the dominant eigenvalue.

🎯 HOW TO APPROACH IT

This is the SAME mechanical loop as the shifted power method from the real exam — just skip the shift step entirely. Multiply, find the largest-magnitude component (that's your eigenvalue estimate), divide the whole vector by it (normalise), repeat.

ITERATE: u = Av, m = largest |component|, v_new = u/m
ku = Avmv (normalised)
1(4.000, 1.000, 1.000)4.000(1.000, 0.250, 0.250)
2(4.500, 1.750, 1.500)4.500(1.000, 0.389, 0.333)
3(4.722, 2.167, 1.667)4.722(1.000, 0.459, 0.353)
4(4.812, 2.377, 1.706)4.812(1.000, 0.494, 0.355)
5(4.848, 2.482, 1.709)4.848(1.000, 0.512, 0.353)

|m₅ − m₄| = |4.848 − 4.812| = 0.036 < 0.05 → converged

✓ FINAL ANSWER

Dominant eigenvalue ≈ 4.85, eigenvector ≈ (1.000, 0.512, 0.353) (true value ≈ 4.87 — a couple more iterations would tighten this further)

🔑 KEY TRICK

Starting from a simple vector like (1,0,0) is completely fine — the power method self-corrects toward the dominant direction regardless of where you start, AS LONG AS your start isn't perfectly perpendicular to the true dominant eigenvector (astronomically unlikely by accident). Never shift unless the question specifically asks for the smallest or an intermediate eigenvalue.

💡 REMEMBER

Plain power method → largest eigenvalue directly. Shifted (A−λ₁I) power method → smallest eigenvalue (then add λ₁ back). If two eigenvalues TIE for largest magnitude, the method fails to converge to a single eigenvector.

PREDICTION 3 — CH.8 INTERPOLATION
PRACTICE QUESTION 3 — predicted: forward-difference interpolation (2025 used backward)

A sensor logs temperature every hour: t = 0,1,2,3,4 → temp (°C) = 10.0, 12.7, 17.2, 24.1, 34.0. Estimate the temperature at t = 0.6 using Newton's forward-difference formula.

🧠 FUNDAMENTAL CONCEPT

With 5 data points there is exactly ONE degree-4 polynomial threading through all of them. Newton's forward formula builds that SAME polynomial as a running correction: start with the crudest guess (just y₀, a flat line), then add a slope correction, then a curvature correction, then finer and finer corrections — each extra term is one more "zoom level" of accuracy. Anchoring at the point closest to your target (here, x₀=0, right next to 0.6) means each correction term is small, so the series converges fast and you don't need many terms.

🎯 HOW TO APPROACH IT

t=0.6 sits right at the START of the table (closest to t=0) — this is exactly when forward difference is the RIGHT choice (mirror image of the exam's backward-difference case, where the target was near the END). Build the difference table, compute r=(x−x₀)/h, apply the formula.

STEP 1 — FORWARD-DIFFERENCE TABLE

Δ¹2.704.506.909.90

Top-row (anchored at x₀=0) values: y₀=10.00, Δy₀=2.70, Δ²y₀=1.80, Δ³y₀=0.60, Δ⁴y₀=0.00

STEP 2 — COMPUTE r AND APPLY THE FORMULA

r = (0.6 − 0)/1 = 0.6

p(0.6) = y₀ + rΔy₀ + [r(r−1)/2]Δ²y₀ + [r(r−1)(r−2)/6]Δ³y₀

= 10.00 + (0.6)(2.70) + [(0.6)(−0.4)/2](1.80) + [(0.6)(−0.4)(−1.4)/6](0.60)

= 10.00 + 1.62 − 0.216 + 0.0336 = 11.4376 (the Δ⁴ term contributes exactly 0, since Δ⁴y₀=0.00 — a strong hint this data actually comes from a cubic function)

✓ FINAL ANSWER

Estimated temperature at t=0.6 ≈ 11.44 °C

🔑 KEY TRICK

If the differences shrink to (near) zero at some order — like Δ⁴≈0 here — the data is (very close to) a polynomial of that order minus one (cubic, in this case). That's a fast way to sanity-check your difference table arithmetic: if the last column ISN'T settling toward something small, you likely made a subtraction error upstream.

💡 REMEMBER

Forward-difference coefficients use (r−1), (r−2)… — MINUS signs. (Backward uses PLUS signs: (r+1), (r+2)…) Best accuracy needs r between 0 and 1 — always anchor at the end of the table nearest your target.

PREDICTION 4 — CH.9 NUMERICAL DIFFERENTIATION
PRACTICE QUESTION 4 — predicted: backward-difference formulas (2025 used forward only)

A function f(x) = xeˣ is tabulated: x = 2.0, 2.1, 2.2, 2.3, 2.4 → f(x) = 14.778, 17.149, 19.855, 22.941, 26.456. Estimate f′(2.4) using h=0.1 with: (i) two-point backward, (ii) three-point backward. Given the exact f′(x)=eˣ(x+1), compute the absolute error for each.

🧠 FUNDAMENTAL CONCEPT

Exactly the same Taylor-series idea as forward difference, just looking in the OPPOSITE direction — essential when your target point is the LAST one in a table and there IS no "next" value to look forward to. Backward difference uses the point itself plus one or more PRECEDING points instead.

🎯 HOW TO APPROACH IT

x=2.4 is the last (rightmost) entry in the table — forward difference is IMPOSSIBLE here (there's no f(2.5) to use), so backward is the only choice. Plug directly into the backward formulas, then compare against the given exact derivative.

▸ Show Full Working▾ Hide Full Working
(i) TWO-POINT BACKWARD

f′(2.4) ≈ [f(2.4) − f(2.3)] / h = [26.456 − 22.941] / 0.1 = 35.150

(ii) THREE-POINT BACKWARD

f′(2.4) ≈ [3f(2.4) − 4f(2.3) + f(2.2)] / (2h) = [79.368 − 91.764 + 19.855] / 0.2 = 37.297

ERROR COMPARISON

Exact: f′(2.4) = e^2.4 (2.4+1) = 11.0232 × 3.4 = 37.479

MethodEstimateExactAbsolute Error
2-point backward35.15037.4792.329
3-point backward37.29737.4790.182

Error shrinks by roughly 13× from 2-point (O(h)) to 3-point (O(h²)) — exactly the accuracy jump you'd expect from one extra data point.

✓ FINAL ANSWER

f′(2.4) ≈ 35.150 (2-point) or 37.297 (3-point) — errors 2.329 and 0.182 respectively

🔑 KEY TRICK

Backward-difference formulas are a MIRROR IMAGE of forward ones — same structure, but every h becomes −h. The 3-point backward formula's coefficients (3, −4, 1) are the exact NEGATIVE-reverse of the 3-point forward's (−3, 4, −1) read backwards — if you know one, you can reconstruct the other under exam pressure.

💡 REMEMBER

Forward → use when your target is at the START of the data (no "previous" point exists). Backward → use when your target is at the END (no "next" point exists). Central → use whenever BOTH neighbours exist; it's always more accurate than either one-sided formula.

PREDICTION 5 — CH.10 NUMERICAL INTEGRATION
PRACTICE QUESTION 5 — predicted: Simpson's 3/8 Rule (on formula sheet, never asked in 2025)

Approximate ∫₁⁴ (1/x) dx using Simpson's 3/8 rule with N=6.

🧠 FUNDAMENTAL CONCEPT

Simpson's 1/3 rule fits a parabola through every GROUP OF 3 points (2 subintervals) — which is why it needs N even. Simpson's 3/8 rule instead fits a CUBIC curve through every GROUP OF 4 points (3 subintervals) — an even better-fitting shape for the same reason a parabola beats a straight line. It exists specifically for when N is a multiple of 3 (or when you have an awkward number of subintervals that don't divide evenly into pairs).

🎯 HOW TO APPROACH IT

N=6 is divisible by 3 (two groups of 3 subintervals: x₀–x₃ and x₃–x₆) — perfect for the 3/8 rule. Compute h and all 7 node values once, then apply the formula's 1,3,3,2,3,3,…,1 coefficient pattern.

STEP 1 — NODES (h = (4−1)/6 = 0.5)
x1.01.52.02.53.03.54.0
f(x)=1/x1.00000.66670.50000.40000.33330.28570.2500
STEP 2 — APPLY THE 3/8 FORMULA

∫ ≈ (3h/8)[(f₀+f₆) + 3(f₁+f₂+f₄+f₅) + 2f₃]

= (3×0.5/8)[(1.0000+0.2500) + 3(0.6667+0.5000+0.3333+0.2857) + 2(0.4000)]

CHECK AGAINST SIMPSON'S 1/3 (same N, for comparison)

Simpson's 1/3 (N=6) on the same integral gives ≈ 1.3877. Exact = ln(4) ≈ 1.3863.

3/8 error ≈ 0.0025  ·  1/3 error ≈ 0.0014 — both excellent, 1/3 marginally better here (the 3/8 rule's real value is enabling EVEN groupings of 3 when 1/3 can't be used at all, not necessarily beating 1/3 head-to-head).

✓ FINAL ANSWER

∫₁⁴ (1/x) dx ≈ 1.3888 using Simpson's 3/8 (N=6) — true value ln(4) ≈ 1.3863

🔑 KEY TRICK

The 3/8 coefficient pattern is 1,3,3,2,3,3,2,…,3,3,1 — endpoints get 1, EVERY OTHER interior node (the ones ending each group of 3) gets 2, all the rest get 3. If N is a multiple of 3 but NOT of 2 (e.g. N=9), you literally cannot use Simpson's 1/3 at all — 3/8 becomes mandatory, not just an option.

💡 REMEMBER

1/3 rule needs N even (groups of 2 subintervals, coefficient 1-4-2-4…-1). 3/8 rule needs N divisible by 3 (groups of 3 subintervals, coefficient 1-3-3-2-3-3…-1). They solve the same "N doesn't divide nicely" problem in different ways.

PREDICTION 6 — CROSS-CHAPTER DISCUSSION
PRACTICE QUESTION 6 — predicted: "compare and justify" discussion sub-question

Compare the Bisection method and the Newton-Raphson method in terms of convergence guarantee and speed. You are asked to find a root of a function whose derivative is very expensive to compute — which method would you recommend, and why? (3–4 marks, written-answer style)

🧠 FUNDAMENTAL CONCEPT

This question isn't testing arithmetic — it's testing whether you understand the TRADE-OFF each method makes. Every root-finding method trades some combination of speed, robustness, and the amount of information it needs about f. Bisection uses the LEAST information (just signs) and is the slowest but unbreakable; Newton uses the MOST information (exact slope) and is the fastest but can fail. Recognising which trade-off fits a given real-world constraint is the actual skill being examined.

🎯 HOW TO APPROACH IT

Structure a written answer in 3 parts: (1) state the convergence guarantee difference, (2) state the speed difference (with the right technical term — linear vs quadratic), (3) directly answer the "which would you recommend" scenario with a reason tied back to the SPECIFIC constraint given (expensive derivative).

▸ Show Full Working▾ Hide Full Working
MODEL ANSWER

Convergence guarantee: Bisection is guaranteed to converge from ANY valid sign-change bracket [a,b] — it can never fail once IVT is satisfied. Newton-Raphson has NO such guarantee: a poor starting guess, a flat tangent (f′(xₙ)≈0), or a sharply curving region can make it diverge entirely.

Speed: Bisection converges linearly — the error is only guaranteed to halve each iteration (a fixed, slow rate regardless of the function). Newton-Raphson converges quadratically near the root — the number of correct digits roughly DOUBLES each iteration, making it dramatically faster once it's in the right neighbourhood.

Recommendation: if f′(x) is very expensive to compute, Bisection is the better choice — Newton-Raphson requires evaluating f′ at EVERY single iteration, so its speed advantage (fewer iterations) is wiped out if each iteration costs far more to compute. In this situation, the Secant method is actually the best middle ground: it approximates the slope using two recent f-values instead of a true derivative, avoiding the expensive f′ calculation entirely while still converging faster than bisection (order ≈1.618 vs bisection's linear rate).

✓ MODEL CONCLUSION

Recommend the Secant method as the practical answer — it sidesteps the expensive-derivative problem that rules out Newton-Raphson, while still beating bisection's speed. If only choosing between the two NAMED methods, Bisection wins here specifically because it never needs f′ at all.

🔑 KEY TRICK

For "compare and recommend" questions, always tie your final recommendation back to the EXACT constraint given in the question (here: "derivative is expensive") — a generic "Newton is fastest so use Newton" answer misses the point of the question and loses marks, even if the general comparison facts you stated were correct.

💡 REMEMBER

Speed ranking: Bisection (linear) < Secant (order ≈1.618) < Newton-Raphson (quadratic). Robustness ranking (reverse): Bisection (guaranteed) > Secant > Newton-Raphson (least guaranteed). No method wins on both — that trade-off IS the concept being tested.
