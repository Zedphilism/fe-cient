CHAPTER 4 — NETWORK LAYER: DATA PLANE
QUESTION 1

Within this chapter, name a layer-3 device.

ANSWER

The router.

💡 REMEMBER

Layer 3 = Network layer = IP addresses. Router lives here.

QUESTION 2

Why is that device in Q1 called a layer-3 device?

ANSWER

Because it forwards packets using Layer-3 (IP) header info — specifically the destination IP address — to decide the output port.

💡 REMEMBER

Hub = L1 (bits) · Switch = L2 (MAC) · Router = L3 (IP).

QUESTION 3

Explain the 2 main functions of the network layer.

ANSWER

1. Forwarding — move a packet input→output inside one router. Fast, local, data-plane (hardware, microseconds).

2. Routing — compute the end-to-end path across the whole network. Slower, network-wide, control-plane (algorithms, seconds). Populates the tables Forwarding uses.

💡 REMEMBER

Routing builds the map; forwarding follows it.

QUESTION 5

If an IP address (200.20.226.5/21) is given, find the subnet's network and broadcast address. Show your workings.

ANSWER

Mask: /21 → 255.255.248.0 (5 bits left in octet 3 → 248)

Block size: 2⁽⁸⁻⁵⁾ = 8 → blocks: …216, 224, 232…

Locate 226: 224 ≤ 226 < 232 → belongs to the 224 block

Network: 200.20.224.0  ·  Broadcast: 224+8−1=231 → 200.20.231.255

Usable range: .224.1 – .231.254 (2046 hosts)

💡 REMEMBER

Block size = 2^(8−mask bits) in the varying octet. Network = round DOWN to the block; Broadcast = network + block − 1.

QUESTION 7

With respect to DHCP, answer the following questions.

💡 REMEMBER

DORA = Discover → Offer → Request → Acknowledge. Renew kicks in at ~50% of the lease (T1) — don't wait for full expiry to matter.

(a) What is DHCP?
ANSWER

Dynamic Host Configuration Protocol — auto-assigns an IP address + config (subnet mask, gateway, DNS) to a host joining the network. No manual setup.

(b) How does DHCP work?
ANSWER

DORA (all broadcast): Discover → Offer → Request → Acknowledge.

(c) What is the benefit of DHCP?
ANSWER

No manual IP assignment, no address conflicts, and efficient reuse of addresses via leases.

(d) What portion of the subnetwork address is used for distribution?
ANSWER

The host portion — a configured pool of host addresses (excluding network, broadcast, and static addresses). The network prefix stays fixed for everyone.

(e) Lease time — what happens on expiry, and if still in use?
ANSWER

The lease is a timed rental. On expiry, the address returns to the pool. An active host renews at ~50% of the lease (T1) — it doesn't wait to expire. If renewal AND rebinding both fail, the host must stop using the address and restart DORA, to avoid a conflict with whoever gets it next.

QUESTION 8

With regards to Network Address Translation (NAT), answer the following questions.

💡 REMEMBER

NAT translates (private IP:port) ↔ (public IP:port). Purpose: save public IPv4 addresses — one public IP for a whole private network.

(a) What is NAT?
ANSWER

Lets a whole private network share one public IP address by translating private ↔ public IP:port at the border router.

(b) What is the purpose of NAT?
ANSWER

Conserve public IPv4 addresses — one org needs only one public IP regardless of internal host count. Bonus: hides internal topology.

(c) How does NAT work?
ANSWER

Router keeps a translation table: (private IP:port) ↔ (public IP:port). Out: rewrite source to public IP:new port, record mapping. In: look up destination port, rewrite back to private IP:port.

QUESTION 9

Given N = 5000, RTT = 150 ms, C = 5 Gbps, how much is average buffer size B?

💡 REMEMBER

Rule of thumb: B = RTT×C. Recent: divide by √N — many flows' bursts smooth each other out, so much less buffer is actually needed.

(a) Following rule of thumb
ANSWER

B = RTT × C = 0.15 s × 5000 Mbps = 750 Mbits (≈ 93.75 MB)

(b) Following recent recommendations
ANSWER

B = (RTT×C)/√N = 750 / √5000 ≈ 10.61 Mbits (≈ 1.33 MB) — about 70× smaller.

QUESTION 10

A 6000-byte file is sent through a link with MTU 1500 bytes. Explain the fragmentation process, using the fragmentation table format from class.

ANSWER

Setup: 6000 bytes = 20-byte header + 5980 data. Max data/fragment = 1500−20 = 1480. Fragments: ⌈5980/1480⌉ = 5.

💡 REMEMBER

Max data = MTU − 20. Offset = start byte ÷ 8. Only the LAST fragment has MF=0. Reassembly = destination only.

QUESTION 12

Why was UDP chosen for SNMP instead of TCP? Explain your answer.

ANSWER

SNMP manages devices that may already be congested or failing — the worst time for a heavyweight protocol. UDP: (1) no handshake to hang on a struggling device, (2) no connection state to maintain, (3) minimal overhead for frequent small polls, (4) the SNMP manager retries lost polls itself — TCP's retransmissions would only add MORE load during congestion.

💡 REMEMBER

A sick network needs a light protocol — no handshake, no state, no extra retransmit overhead.
