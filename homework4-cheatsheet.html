CHAPTER 4 — NETWORK LAYER: DATA PLANE
QUESTION 1

Within this chapter, name a layer-3 device.

ANSWER

The router.

QUESTION 2

Why is that device in Q1 called a layer-3 device?

ANSWER

Because it operates at the network layer (layer 3) of the OSI/TCP-IP stack — it makes its forwarding decisions by examining layer-3 (IP) header information, specifically the destination IP address, to decide which output port a packet should be sent to. This is different from a switch (layer 2, uses MAC addresses) or a hub (layer 1, just repeats bits).

QUESTION 3

Explain the 2 main functions of the network layer.

ANSWER

1. Forwarding — moving a packet from an input link to the correct output link inside a single router. This is a local, per-router, data-plane action that happens in hardware within microseconds: look up the destination in the forwarding table, switch the packet to the right output port.

2. Routing — determining the end-to-end path a packet takes from source to destination across the whole network. This is a network-wide, control-plane process (routing algorithms like Dijkstra or Bellman-Ford) that runs on the scale of seconds and populates the forwarding tables that Function 1 then uses.

Routing computes the tables; forwarding uses them.

QUESTION 5

If an IP address (200.20.226.5/21) is given, find the subnet's network and broadcast address. Show your workings.

ANSWER

Step 1 — subnet mask: /21 → 255.255.248.0 (21 network bits: 8+8+5 = 21, so octet 3 keeps 5 mask bits = 11111000 = 248)

Step 2 — block size in octet 3: 2⁽⁸⁻⁵⁾ = 2³ = 8 → subnets fall on multiples of 8 in the third octet: …, 216, 224, 232, …

Step 3 — locate 226: 224 ≤ 226 < 232, so this address belongs to the 224 block.

Network address: 200.20.224.0

Broadcast address: 224 + 8 − 1 = 231 → 200.20.231.255

Usable host range: 200.20.224.1 – 200.20.231.254 (2¹¹ − 2 = 2046 usable hosts)

QUESTION 7

With respect to DHCP, answer the following questions.

(a) What is DHCP?
ANSWER

Dynamic Host Configuration Protocol — a protocol that allows a host to automatically obtain an IP address and other network configuration (subnet mask, default gateway, DNS server) when it joins a network, without any manual setup ("plug-and-play").

(b) How does DHCP work?
ANSWER

Through the DORA four-message exchange (all broadcast initially): Discover (client broadcasts a request) → Offer (one or more DHCP servers respond with an offered IP + config) → Request (client broadcasts acceptance of one offer) → Acknowledge (the chosen server confirms and the lease begins).

(c) What is the benefit of DHCP?
ANSWER

Automatic, centrally-managed configuration — no manual IP assignment needed, no risk of two hosts accidentally getting the same address, and addresses are reused efficiently via leases as devices join and leave the network.

(d) What portion of the subnetwork address is used for distribution?
ANSWER

The host portion of the subnet — specifically, an administrator-configured pool/range of host addresses within that portion (e.g. .100–.200), excluding the network address, broadcast address, and any statically-assigned addresses (gateway, servers, printers). The network prefix stays fixed for everyone on the subnet; only host bits vary between clients.

(e) Lease time — what happens on expiry, and if it expires while still in use?
ANSWER

The lease is a timed rental of the address (a common example duration is 24 hours, though it's configurable). On expiry, the address returns to the pool for reuse by another client. An actively-using host doesn't just wait for expiry — at roughly the halfway point of the lease (T1, ~50%) it unicasts a renewal Request to the server to extend the lease invisibly. If renewal fails and rebinding (a broader retry at ~87.5% of the lease) also fails, and the lease truly lapses, the host must stop using the address and restart the DORA discovery process from scratch — otherwise it risks an IP address conflict with whoever the server assigns that address to next.

QUESTION 8

With regards to Network Address Translation (NAT), answer the following questions.

(a) What is NAT?
ANSWER

A technique that lets an entire private network share a single public IP address by translating between private (internal) and public (external) IP address/port combinations at the border router.

(b) What is the purpose of NAT?
ANSWER

To conserve the shrinking IPv4 public address space — one organisation only needs to purchase/lease one public IP no matter how many internal hosts it has. It also has the side benefit of hiding the internal network's topology and addressing from the outside.

(c) How does NAT work?
ANSWER

The NAT router maintains a translation table mapping (private IP, private port) ↔ (public IP, public port) for every active connection.

Outgoing packet: the router replaces the source private IP:port with its own public IP and a newly-assigned port, and records this mapping in the table.

Incoming packet: the router looks up the destination port in the translation table, and rewrites the destination back to the correct private IP:port before forwarding it onto the internal network.

QUESTION 9

Given N = 5000, RTT = 150 ms, C = 5 Gbps, how much is average buffer size B?

(a) Following rule of thumb
ANSWER

Formula: B = RTT × C

B = 0.15 s × 5000 Mbps = 750 Mbits (≈ 93.75 MB / 93.75 megabytes)

(b) Following recent recommendations
ANSWER

Formula: B = (RTT × C) / √N

B = 750 Mbits / √5000 = 750 / 70.71 ≈ 10.61 Mbits (≈ 1.33 MB)

With many independent TCP flows (N large), their traffic bursts statistically de-synchronise and smooth out, so far less buffering is needed than the conservative rule-of-thumb suggests — about 70× less in this case.

QUESTION 10

A 6000-byte file is sent through a link with MTU 1500 bytes. Explain the fragmentation process, using the fragmentation table format from class.

ANSWER

Setup: total datagram = 6000 bytes = 20-byte IP header + 5980 bytes of data. MTU = 1500 bytes → max data per fragment = 1500 − 20 = 1480 bytes.

Number of fragments: ⌈5980 / 1480⌉ = ⌈4.04⌉ = 5 fragments

QUESTION 12

Why was UDP chosen for SNMP instead of TCP? Explain your answer.

ANSWER

SNMP (Simple Network Management Protocol) manages devices that may already be congested, failing, or under stress — exactly when a heavyweight, stateful protocol is least appropriate. UDP is chosen because:

1. No connection overhead — no 3-way handshake to fail or hang on a struggling device.

2. No connection state — the router/switch being managed doesn't need to keep per-connection memory alive for the manager, which matters on resource-constrained network equipment being polled by many managers.

3. Minimal overhead for frequent small polls — SNMP traffic is typically short polling messages sent repeatedly; TCP's setup/teardown cost would dominate.

4. Application-level retry is sufficient — if a poll is lost, the SNMP manager itself simply retries; it doesn't need TCP's automatic retransmission, and under congestion TCP's retransmissions would add MORE load exactly when the network can least afford it.
