QUESTION 1 — MULTIPLEXER, PARALLEL ADDER & TRUE/FALSE
QUESTION 1 [15 MARKS]

a) Figure 1 shows a logic diagram for a 4-input multiplexer. (i) Create the truth table. (ii) Draw the output waveform for the given timing diagram.

b) Using a parallel adder, add binary 1011 and 0101. (i) Draw and label the block diagram. (ii) Calculate the sum, carry-in and output carry.

c) Identify each statement as TRUE or FALSE (5 statements on adders, XOR parity, comparators and demultiplexers).

🧠 FUNDAMENTAL CONCEPT — start here if any of this is new to you

What a multiplexer (MUX) actually does: imagine four people (D0, D1, D2, D3) all want to talk into the same single microphone (Y), but only one can speak at a time. A MUX is the "switchboard operator" — the SELECT lines (S0, S1) act like a 2-bit address that says "connect input number ___ to the output right now", and everyone else is silenced. With 2 select lines you can address 4 distinct inputs (2²=4), which is exactly why a 4-input MUX needs exactly 2 select lines.

Why it's built from AND/OR gates: each data input Dᵢ is fed into its own AND gate, together with the UNIQUE combination of S1/S1' and S0/S0' that is true only when the select lines spell out that input's address (e.g. D2's AND gate uses S1·S0'). Since only ONE address can be "spelled out" at any instant, only ONE AND gate can be open at a time — all the others output 0 automatically. The final OR gate simply merges these four AND-gate outputs; since at most one is ever non-zero, the OR gate effectively just "passes through" whichever single input was selected.

What a parallel adder is (part b): adding two 4-bit numbers means adding 4 separate columns of bits, just like adding by hand on paper — except each column's addition can generate a "carry" into the NEXT column, exactly like carrying a 1 when you add 7+5 by hand. A parallel adder is simply 4 full adders chained together, where each stage's carry-OUT feeds directly into the next stage's carry-IN — the whole 4-bit sum is produced in one pass, with the carries rippling through the chain.

Why part (c) matters: these five statements test whether you actually understand WHAT each circuit (adder, XOR-parity chain, comparator, demultiplexer) does, not just how to build it — a fast, reliable way exam-setters check real understanding without demanding a full derivation.

🎯 HOW TO APPROACH IT

(a)(i) Read the AND-gate wiring directly off Figure 1: gate 1 uses S1'S0' (selects D0), gate 2 uses S1'S0 (selects D1), gate 3 uses S1S0' (selects D2), gate 4 uses S1S0 (selects D3). Y always just equals whichever D the current S1S0 pair addresses.

(a)(ii) At every time-slice, read off the CURRENT values of S1 and S0, use them to pick which Dᵢ is "connected", and copy THAT signal's current value onto Y for that slice.

(b) Add bit by bit from the LSB, carrying into the next column exactly like column addition on paper. Cin for bit 0 is 0 unless the question states otherwise.

(c) Check each statement against the exact formal definition of that component — watch for subtly wrong numbers ("two inputs" vs the real count) and swapped comparisons (A<B vs A>B).

QUESTION 2 — LATCHES, FLIP-FLOPS & ASYNCHRONOUS COUNTER
QUESTION 2 [16 MARKS]

a) Draw the output waveform for a gated D latch given D and EN timing diagrams.

b) Table 1 gives 8 rows of PRE', CLR', J, K for a negative-edge JK flip-flop. Determine output Q and its state for each row.

c) A circuit has a JK flip-flop (FF1) and D flip-flop (FF2), both negative-edge, plus an AND gate producing output Y (Figure 4, "Asynchronous counter"). (i) Write equations for D and Y. (ii) Simulate Table 2's 8 rows for Q1, Q2 and Y.

🧠 FUNDAMENTAL CONCEPT — start here if any of this is new to you

What a latch/flip-flop fundamentally is: it's a tiny piece of MEMORY — one single bit that can be SET to 1, CLEARED to 0, or HELD at its current value, and it keeps remembering that value even after the signal that set it disappears (unlike a plain logic gate, whose output only exists while its inputs are actively held). This "remembering" is what lets digital circuits have history — a counter, a register, anything that needs to track "what happened before" needs flip-flops, not just gates.

Latch vs flip-flop — the key difference: a latch is "transparent" — while its enable (EN) is HIGH, its output Q follows the D input immediately and continuously; the instant EN drops LOW, Q freezes at whatever D was at that exact moment. A flip-flop is stricter — it only looks at its input for one infinitesimal instant (the rising or falling EDGE of a clock pulse), and ignores its input completely at every other moment. This is why part (a) draws a continuous waveform (Q tracks D whenever EN=1) while part (b) only updates Q at discrete falling-edge moments.

What J and K actually control: a JK flip-flop is a "smarter" SR flip-flop that fixes the one broken case (S=R=1) by making it mean "toggle" instead of "invalid". J=0,K=0 → hold (remember). J=1,K=0 → Set (Q=1). J=0,K=1 → Reset (Q=0). J=1,K=1 → Toggle (flip whatever Q currently is) — this toggle behaviour is exactly what makes JK flip-flops the natural building block for counters (a bit that flips every clock tick is precisely what a binary counter bit does).

Why PRE' and CLR' exist, and why they're called "asynchronous": sometimes you need to force a flip-flop to a known value RIGHT NOW, without waiting for the next clock edge — e.g. to reset a whole circuit to 0 at power-on. PRE' (active-LOW preset) and CLR' (active-LOW clear) are override inputs that punch straight through to Q the INSTANT they go LOW, completely ignoring the clock and J/K — that's what "asynchronous" means here: not tied to the clock's schedule at all. They always take priority over the normal, clock-synchronised J/K behaviour.

What makes Q2(c)'s circuit "asynchronous" as a whole: connecting a SECOND flip-flop's clock input to the FIRST flip-flop's OUTPUT (instead of to the master clock directly) creates a ripple effect — each stage only updates after it "feels" the previous stage change, one stage triggering the next like dominoes, rather than everything updating at the exact same master-clock instant. That domino-style, one-thing-triggers-the-next design is what "asynchronous" (ripple) counter means, as opposed to a "synchronous" counter where every flip-flop shares literally the same clock wire.

🎯 HOW TO APPROACH IT

(a) Whenever EN=1, redraw D exactly onto Q. The instant EN drops to 0, freeze Q at whatever value D had at that exact instant, and hold it flat until EN rises again.

(b) For every row, FIRST check PRE'/CLR' — if either is 0, that override wins immediately, no need to even look at J/K. Only when PRE'=CLR'=1 do you apply the normal JK rule at that row's falling clock edge.

(c)(i) Since Table 2 only varies PRE'/CLR' (no J/K columns), FF1's J and K must be fixed — tied HIGH — making FF1 a plain toggle stage. The natural, standard way to make a D flip-flop also toggle is to wire its own D input to its own inverted output, D=Q2'; combined with an AND gate reading both outputs, Y=Q1·Q2.

(c)(ii) Apply PRE'/CLR' overrides first (they hit BOTH flip-flops at once here); otherwise FF1 always toggles, and FF2 only toggles on rows where Q1 has just gone HIGH→LOW (that's its ripple clock edge).

QUESTION 3 — MOD-6 ASYNCHRONOUS COUNTER DESIGN
QUESTION 3 [15 MARKS]

a) Design and draw a logic circuit for a MOD-6 count-up asynchronous counter using negative edge-triggered JK flip-flops with asynchronous inputs.

b) Draw the waveform for the counter outputs for 10 clock cycles, given PRE' and CLR' are HIGH throughout.

🧠 FUNDAMENTAL CONCEPT — start here if any of this is new to you

Why "MOD-6" needs 3 flip-flops: each flip-flop can represent 2 possible values (0 or 1), so n flip-flops together can represent 2ⁿ distinct combinations. To count 6 distinct values (0 through 5), you need the smallest n where 2ⁿ ≥ 6 — that's n=3 (2³=8). You'll always have "spare" unused combinations (here, 6 and 7) that the design must actively avoid landing on.

Why a plain 3-bit binary ripple counter isn't enough on its own: left alone, 3 toggling flip-flops naturally count 0,1,2,3,4,5,6,7,0,... (all 8 combinations, a MOD-8 counter). To force it to "wrap around" after 5 instead of 7, you need to actively DETECT the moment the count reaches 6 and immediately snap it back to 0 — before the circuit has a chance to settle into that unwanted state.

How the detection-and-reset trick works: feed the two bits that are HIGH exactly at count=6 (binary 110, i.e. Q2 and Q1) into a NAND gate, and wire that NAND gate's output into the CLR' (active-low clear) pins of every flip-flop. The instant the count would become 110, the NAND output drops LOW, asynchronously slamming every flip-flop back to 000 — faster than the circuit can even finish "settling" into 110, so in practice you never actually observe state 6 on a waveform; you just see 5 jump straight back to 0.

🎯 HOW TO APPROACH IT

(a) Use the standard "detect-and-clear" recipe: 3 negative-edge JK flip-flops, all J=K=1 (toggle), ripple-connected (Q0 clocks FF1, Q1 clocks FF2), with a 2-input NAND gate reading Q2 and Q1 and feeding every flip-flop's CLR' pin.

(b) Simulate the natural binary count 0→7 but snap back to 0 the instant you would reach 6 — the waveform simply repeats 0,1,2,3,4,5 over and over.

(a) CIRCUIT DESIGN

3 negative-edge JK flip-flops (FF0=Q0 LSB, FF1=Q1, FF2=Q2 MSB), every J and K tied HIGH (=1) so each stage toggles on its own active clock edge. Ripple wiring: master CLK → FF0's clock; Q0 → FF1's clock; Q1 → FF2's clock (each stage clocked by the PREVIOUS stage's output, the hallmark of an asynchronous/ripple counter).

Reset logic: a 2-input NAND gate with inputs Q2 and Q1, output wired to ALL THREE flip-flops' CLR' pins. NAND(Q2,Q1) is LOW only when Q2=Q1=1 — which first happens at count 110 (decimal 6) — instantly clearing the count back to 000.

🔑 KEY TRICK

For any MOD-N asynchronous counter design question: (1) find the smallest n with 2ⁿ≥N, (2) write N in binary, (3) feed EVERY bit that's a "1" in that binary pattern into a NAND gate, (4) wire that NAND's output to every flip-flop's CLR'. This recipe works for ANY modulus — MOD-5 needs Q2,Q0 (101), MOD-10 needs 4 flip-flops with Q3,Q1 (1010), and so on.

💡 REMEMBER

The NAND-into-CLR' trick always detects the FIRST unwanted state (N itself, not N−1) and resets before it can be observed — the counter's visible range is always 0 to N−1.

QUESTION 4 — SHIFT REGISTER & JOHNSON COUNTER
QUESTION 4 [14 MARKS]

a) Figure 5 shows a universal bidirectional shift register timing diagram (current parallel input = decimal 3). (i) List the basic data transfers a shift register can perform. (ii) Complete the timing diagram for the parallel outputs.

b) Figure 6 shows a 3-bit Johnson counter. (i) Construct a complete truth table for 8 clock cycles, starting at decimal 4. (ii) Draw the state diagram.

🧠 FUNDAMENTAL CONCEPT — start here if any of this is new to you

What a shift register is: picture a row of lockers, each holding one bit, connected in a line. Every clock pulse, every bit "steps sideways" into its neighbour's locker, and one NEW bit enters at the end of the line. A "universal" shift register additionally supports loading all lockers at once directly (parallel load) instead of shuffling one bit in at a time — giving it four possible data-transfer modes depending on how you get data IN and how you read data OUT: SISO (serial in, serial out), SIPO (serial in, parallel out), PISO (parallel in, serial out), PIPO (parallel in, parallel out).

What a Johnson counter is, and why it's "twisted": take a plain ring counter (a shift register whose LAST output is wired straight back to its FIRST input, endlessly recirculating a single 1) — a Johnson counter makes one small but powerful change: it feeds back the INVERTED last output instead of the plain one. That single inversion means the pattern doesn't just repeat after n steps like a ring counter — it takes 2n steps to return to its starting value, because the circuit has to complete a full "all 1s" phase AND a full "all 0s" phase before repeating. For 3 bits, that's 2×3=6 distinct states, compared to a plain ring counter's only 3.

🎯 HOW TO APPROACH IT

(a)(i) This is a definition-recall question — list all four transfer modes by name and one-line meaning.

(a)(ii) Watch for the SH/CLK' line dropping LOW — that's the signal to parallel-load D0,D1,D2 straight into Q0,Q1,Q2 for that instant, overriding the normal serial shifting.

(b) Apply the Johnson shift rule every clock: Q0_new = NOT(Q2_old), Q1_new = Q0_old, Q2_new = Q1_old — then just repeat 8 times from the given starting value.

QUESTION 5 — SYNCHRONOUS COUNTER FROM A GATE-LEVEL CIRCUIT
QUESTION 5 [20 MARKS]

Refer to the circuit in Figure 7 (a synchronous counter built from two positive-edge JK flip-flops plus combinational gates, with input X and output Y). i. Derive Boolean expressions for J1, K1, J0, K0 and Y. ii. Produce the next-state and flip-flop transition table. iii. Draw the complete state diagram.

Trace your own figure's wires into each J/K pin the same way and the method transfers directly.

🧠 FUNDAMENTAL CONCEPT — start here if any of this is new to you

Synchronous vs asynchronous, revisited: in Q2/Q3 the flip-flops were clocked one after another (ripple). Here, EVERY flip-flop shares the exact same clock wire — they all sample their J/K inputs and update Q at the SAME instant. Because there's no rippling delay, synchronous counters can run much faster, but the trade-off is that the combinational logic feeding each J/K input must be cleverly designed in advance so that the RIGHT thing happens simultaneously in every stage.

What "deriving the equations" really means: J1, K1, J0, K0 aren't free choices — they're the OUTPUTS of ordinary logic gates (AND/OR/NOT) whose INPUTS are the counter's own current state (Q1, Q0) and the external enable X. To "derive" them just means: trace which wires feed into each flip-flop's J and K pins, and write down the Boolean expression those gates compute.

Why an "enable" input matters: a bare counter always counts every clock tick with no way to pause it — almost useless in a real system where you sometimes need to freeze the count (e.g. while reading its value). Adding an enable input X into every J/K expression (as a multiplying factor) means "only allow toggling when X=1"; when X=0, every J and K collapses to 0, which is exactly the JK "hold" condition — so the whole counter freezes in place.

🎯 HOW TO APPROACH IT

Trace each labelled wire in the diagram back to the gate(s) that produce it, write the Boolean expression each gate computes, then build a standard present-state/next-state table by applying the JK truth table (00=hold,01=reset,10=set,11=toggle) to each derived J/K pair. Read the state diagram directly off the completed table — every present-state row becomes one arrow to its next-state.

🔑 KEY TRICK

For any synchronous binary up-counter, the pattern always generalises: bit-i's J and K both equal "enable AND every lower bit is currently 1" — bit 0 needs just the enable, bit 1 needs enable·Q0, bit 2 needs enable·Q1·Q0, and so on (the classic ripple-carry pattern, but computed synchronously in parallel instead of rippling).

To derive J/K equations from ANY gate-level counter diagram: trace each J and K pin back to its feeding gate(s) and write the Boolean expression — then build the table by applying the ordinary JK rule (00 hold / 01 reset / 10 set / 11 toggle) row by row.

QUESTION 6 — DESIGNING A COUNTER FROM A STATE DIAGRAM (T FLIP-FLOPS)
QUESTION 6 [20 MARKS]

Refer to the state diagram in Figure 8 (states 7↔2↔3↔6 forming a loop, arcs labelled X/Z). Design a synchronous counter using T flip-flops with input X and output Z. a) Construct a complete next-state and T flip-flop transition table. b) Draw K-maps to get the minimum SOP Boolean expression. c) Draw the complete circuit diagram.

🧠 FUNDAMENTAL CONCEPT — start here if any of this is new to you

What a T flip-flop is: the simplest possible flip-flop behaviour — just ONE input, T. T=0 means hold (do nothing), T=1 means toggle (flip Q). It's literally the JK flip-flop with J and K permanently tied together (so they're always equal, collapsing the 4 JK cases down to just these two useful ones) — the natural choice whenever a bit's ENTIRE job is "sometimes flip, otherwise stay put", which is exactly what every bit inside a counter needs to do.

What "designing FROM a state diagram" means, step by step: a state diagram is the SPEC — it tells you every state, and for every possible input, exactly which state comes next and what output to produce. Turning that spec into a real circuit is mechanical: (1) assign binary codes to every state, (2) for every state+input combination read off the required next state, (3) for each bit, ask "did this bit need to change (T=1) or stay the same (T=0)?" between present and next state, (4) once you have T as a function of the present state and input for every flip-flop, simplify that function with a K-map, (5) build the AND/OR/NOT gates that compute the simplified expression and wire them into each flip-flop's T pin.

What a K-map actually does: a truth table lists every input combination in a straight line, which hides which ROWS are "close relatives" of each other (differing in only 1 bit). A Karnaugh map instead arranges the same rows in a grid where every physically ADJACENT cell differs by exactly one variable — so any block of adjacent 1s you can visually circle represents a simplification (one variable is irrelevant across that whole block and can be dropped from the expression). It's simplification-by-eye instead of algebra-by-hand.

🎯 HOW TO APPROACH IT

Read every arc directly off Figure 8: "0/1" means X=0 produces output Z=1 and moves forward around the loop (7→2→3→6→7); "1/0" means X=1 produces Z=0 and moves backward (7→6→3→2→7). Convert every state to its 3-bit binary code, then for every row work out T2,T1,T0 by comparing present vs next state bit-by-bit (T=1 exactly where the bit flips). Only 4 of the 8 possible 3-bit codes are ever used — the other 4 are "don't cares" you can exploit freely when simplifying.

Grouping the T2=1 rows: (Q0=1, X=0) and (Q0=0, X=1) — this is exactly the pattern of an XOR gate! T2=1 whenever X and Q0 DISAGREE. Grouping the T2=0 rows confirms it: (Q0=1,X=1) and (Q0=0,X=0) — X and Q0 AGREE. So T2 = X'Q0 + XQ0' = X⊕Q0, and remarkably Q2 itself never appears in the simplified expression — a genuine simplification the K-map below makes visually obvious.

🔑 KEY TRICK

Whenever a state diagram uses only SOME of the 2ⁿ possible codes, immediately mark the unused combinations as don't-cares in every K-map — they're free simplification fuel, and this exact question only becomes clean (T1=0, T0=1 constants) BECAUSE the 4 unused codes were exploited. Always scan your finished T/J/K table for an all-0 or all-1 column first — it means that flip-flop needs NO gate at all, just a wire to ground or Vcc.

💡 REMEMBER

T flip-flop = JK with J tied to K. Designing from a state diagram is always the same 5-step recipe: assign codes → build the transition table → extract T (or J/K) per bit → K-map simplify → wire the gates.
