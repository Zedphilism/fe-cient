SECI1113 — PROBLEM-SOLVING REFERENCE

Step-by-Step Solving Guide

Every numerical method in Chapters 5–10, broken into a fixed, repeatable procedure you can apply to ANY problem of that type — not just the one example shown. Follow the numbered steps in order; skipping a step is the most common way marks are lost.

How to use this page: when you meet a new problem in the exam, first identify WHICH method it's asking for (the question usually names it, or the data shape gives it away — e.g. a sign-change bracket → bisection; equally-spaced table → interpolation). Then follow that method's steps mechanically. The steps never change — only the numbers do.

CHAPTER 5 — ACCURACY & ERROR
METHOD 1
Absolute & Relative Error

Use when: you're given (or can find) a true value and an approximation, and asked "how accurate is this?"

1Identify the true value N and the approximation n.
2Absolute error: |e| = |N − n|.
3Relative error: e_rel = |N − n| / |N| (use |n| in the denominator if N is unknown).
4Percentage error (if asked): e_rel × 100%.
MINI EXAMPLE

N = π ≈ 3.14159265, n = 3.14. |e| = 0.00159. e_rel = 0.00159/3.14159 ≈ 0.00051 ≈ 0.05%.

💡 REMEMBER

Same absolute error can mean excellent or terrible accuracy — it depends on the size of N. Always check relative error when comparing two approximations.

METHOD 2
Rounding Error Propagation

Use when: several rounded numbers are added/subtracted and you must state how many decimal places the final answer can be trusted to.

1For each term rounded to n decimal places, its error bound is 0.5 × 10⁻ⁿ.
2Add ALL the bounds together (triangle inequality) — this works for both addition AND subtraction.
3Compute the arithmetic result normally.
4Report the answer only to the decimal place where (result − total bound) and (result + total bound) still agree.
MINI EXAMPLE

3.69(2dp) + 5.432(3dp) − 2.37(2dp) − 3.5214(4dp) = 3.2306. Bounds: 0.005+0.0005+0.005+0.00005 = 0.01055. Range 3.220–3.241 → both agree only to 1dp → report 3.2.

💡 REMEMBER

The least precise term (fewest decimal places) dominates the final answer's precision — adding more precise terms doesn't rescue it.

CHAPTER 6 — NON-LINEAR EQUATIONS
METHOD 3
Intermediate Value Theorem (IVT) Check

Use when: you must verify a root exists in an interval BEFORE running bisection (always the first step of any bisection question).

1Confirm f is continuous on [a, b] (polynomials always are).
2Compute f(a) and f(b).
3Compute the product f(a) · f(b).
4Product < 0 → root guaranteed in (a,b). Product > 0 → no conclusion. Product = 0 → an endpoint IS the root.
MINI EXAMPLE

f(x)=x³−x−1 on [1,2]: f(1)=−1, f(2)=5. Product = −5 < 0 → root exists in (1,2).

💡 REMEMBER

IVT only proves a root exists — it never says how many, or where. "No conclusion" (product > 0) does NOT mean "no root".

METHOD 4
Bisection Method

Use when: you have a sign-change bracket [a,b] and need a guaranteed (if slow) root-finding method.

1Verify IVT: f(a)·f(b) < 0 (Method 3).
2(If asked) plan iterations: n ≥ log₂((b−a)/ε).
3Compute midpoint c = (a+b)/2 and evaluate f(c).
4If f(a)·f(c) < 0 → new bracket [a, c]. Otherwise → new bracket [c, b].
5Repeat steps 3–4 until the interval width < ε (or f(c) ≈ 0).
MINI EXAMPLE

f(x)=x²−3 on [1,2]. c=1.5, f(1.5)=−0.75 (same sign as f(1)) → new bracket [1.5, 2]. Repeat.

💡 REMEMBER

Always tabulate a, b, c, f(c), new bracket at every iteration — markers award the table, not just the final number.

METHOD 5
Newton-Raphson Method

Use when: f′(x) is easy to compute and you have one good starting guess close to the root — fastest convergence.

1Write f(x) and derive f′(x).
2Choose starting guess x₀.
3Apply xₙ₊₁ = xₙ − f(xₙ)/f′(xₙ).
4Check |xₙ₊₁ − xₙ| < ε — if not satisfied, repeat step 3 with the new x.
MINI EXAMPLE

f(x)=x³−2x²−5, x₀=2: f(2)=−5, f′(2)=4. x₁ = 2−(−5/4) = 3.25. Repeat until convergence.

💡 REMEMBER

Fails if f′(xₙ)=0 (horizontal tangent) or if x₀ is far from the root. No guarantee like bisection — but roughly doubles correct digits every step.

METHOD 6
Secant Method

Use when: the derivative is hard/impossible to compute, but you have two starting guesses.

1Choose two starting guesses x₀, x₁ (no sign-change bracket required).
2Compute f(x₀) and f(x₁).
3Apply x₂ = x₁ − f(x₁)(x₁−x₀) / (f(x₁)−f(x₀)).
4Shift the window: old x₁ becomes new x₀, new x₂ becomes new x₁. Repeat until |xₙ₊₁−xₙ| < ε.
MINI EXAMPLE

f(x)=x³−2, x₀=1, x₁=2: f(1)=−1, f(2)=6. x₂ = 2 − 6(1)/7 ≈ 1.1429. Repeat with (x₁,x₂).

💡 REMEMBER

Always use the TWO MOST RECENT points — the oldest point is discarded every step. Speed order: bisection < secant < Newton.

CHAPTER 7 — EIGENVALUES & EIGENVECTORS
METHOD 7
Eigenvalues via Characteristic Polynomial

Use when: asked to find ALL eigenvalues of a small (2×2 or 3×3) matrix exactly.

1Form A − λI (subtract λ from every diagonal entry).
2Compute det(A − λI) = 0 — this is the characteristic polynomial.
3For 2×2: shortcut is λ² − (trace)λ + det = 0. Solve by factoring or the quadratic formula.
4Each root λᵢ is an eigenvalue. Check: sum of roots = trace(A), product of roots = det(A).
MINI EXAMPLE

A=[[3,6],[1,4]]: λ²−7λ+6=(λ−6)(λ−1)=0 → λ=6, λ=1. Check: 6+1=7=trace ✓, 6×1=6=det ✓.

💡 REMEMBER

An n×n matrix ALWAYS has exactly n eigenvalues (counting repeats/complex ones). Trace+det shortcut solves any 2×2 in one line.

METHOD 8
Eigenvectors (given an eigenvalue)

Use when: you already have λ (from Method 7) and need its corresponding eigenvector.

1Substitute λᵢ into (A − λᵢI)v = 0.
2Row-reduce (Gaussian elimination) the resulting matrix.
3Express the solution in terms of a free parameter t.
4Verify: Av = λv using your found vector.
MINI EXAMPLE

λ=6 for A=[[3,6],[1,4]]: (A−6I)v=0 → [[-3,6],[1,-2]]v=0 → row-reduce → v=t(2,1). Eigenvector: (2,1).

💡 REMEMBER

Eigenvectors are reported "up to scale" — any nonzero multiple is equally correct. Always verify with Av=λv before moving on.

METHOD 9
Power Method

Use when: you only need the DOMINANT (largest) eigenvalue, especially for a large matrix.

1Choose any nonzero starting vector v⁽⁰⁾.
2Compute u⁽¹⁾ = A v⁽⁰⁾.
3m₁ = the largest-magnitude component of u⁽¹⁾ — this IS the current eigenvalue estimate.
4Normalise: v⁽¹⁾ = u⁽¹⁾ / m₁.
5Repeat steps 2–4 until |mₖ₊₁ − mₖ| < ε.
MINI EXAMPLE

A=[[2,1],[1,2]], v⁽⁰⁾=(1,0): u⁽¹⁾=(2,1), m₁=2, v⁽¹⁾=(1,0.5). Repeat — converges toward λ=3.

💡 REMEMBER

Fails if two eigenvalues TIE for largest magnitude. Normalising does two jobs at once: prevents overflow AND gives you mₖ (the eigenvalue estimate).

CHAPTER 8 — INTERPOLATION
METHOD 10
Newton Forward-Difference Interpolation

Use when: estimating a value near the BEGINNING of an equally-spaced table.

1Confirm the x-values are equally spaced with step h.
2Build the forward-difference table: Δyₖ = yₖ₊₁−yₖ, then Δ², Δ³…
3Compute r = (x − x₀) / h.
4Apply pₙ(x) = y₀ + rΔy₀ + [r(r−1)/2!]Δ²y₀ + … using the TOP row of the table.
5Add terms until they become negligible (usually 3–4 terms).
MINI EXAMPLE

y₀=0.5, Δy₀=−0.0455, r=0.5: p≈0.5+(0.5)(−0.0455)=0.4773 (2-term estimate).

💡 REMEMBER

Forward formula coefficients use (r−1), (r−2)… — minus signs. Best accuracy needs r between 0 and 1.

METHOD 11
Newton Backward-Difference Interpolation

Use when: estimating a value near the END of an equally-spaced table.

1Confirm equal spacing; identify xₙ (the LAST point in the table).
2Build the backward-difference table: ∇yₖ = yₖ−yₖ₋₁, then ∇², ∇³…
3Compute r = (x − xₙ) / h (typically negative).
4Apply pₙ(x) = yₙ + r∇yₙ + [r(r+1)/2!]∇²yₙ + … using the BOTTOM row of the table.
MINI EXAMPLE

yₙ=0.3333, ∇yₙ=−0.0238, r=−0.5: p≈0.3333+(−0.5)(−0.0238)=0.3452.

💡 REMEMBER

Backward formula coefficients use (r+1), (r+2)… — PLUS signs (opposite of forward). Mixing the two sign patterns is the #1 exam slip.

CHAPTER 9 — NUMERICAL DIFFERENTIATION
METHOD 12
Forward & Backward Difference (2-point)

Use when: estimating f′(x) at a BOUNDARY point (only one neighbour available).

1Identify h and whether x sits at the left edge (only a "next" point) or right edge (only a "previous" point) of the data.
2Left edge → forward: f′(x) ≈ [f(x+h) − f(x)] / h.
3Right edge → backward: f′(x) ≈ [f(x) − f(x−h)] / h.
MINI EXAMPLE

f(2.0)=0.693, f(2.1)=0.742, h=0.1: forward f′(2.0)≈(0.742−0.693)/0.1=0.49.

💡 REMEMBER

Both are O(h) — halving h only halves the error. Use central difference (Method 13) whenever both neighbours exist.

METHOD 13
Central Difference (3-point)

Use when: x is an INTERIOR point (both neighbours x−h and x+h exist) — always preferred over one-sided formulas.

1Confirm both f(x−h) and f(x+h) are available in the table.
2Apply f′(x) ≈ [f(x+h) − f(x−h)] / (2h).
3Note: f(x) itself is NOT used in the formula.
MINI EXAMPLE

f(2.0)=0.693, f(2.2)=0.788, h=0.1: f′(2.1)≈(0.788−0.693)/0.2=0.475 (more accurate than forward/backward).

💡 REMEMBER

O(h²) — halving h QUARTERS the error (vs only halving for one-sided). But h too small → rounding error takes over. There's an optimal h, not "smaller is always better".

CHAPTER 10 — NUMERICAL INTEGRATION
METHOD 14
Trapezoidal Rule

Use when: approximating ∫f(x)dx and N can be ANY value (odd or even).

1Compute h = (b−a)/N.
2List nodes x₀…xₙ and compute f at each node.
3Apply (h/2)[f₀ + 2(f₁+f₂+…+fₙ₋₁) + fₙ] — endpoints ×1, interior nodes ×2.
MINI EXAMPLE

∫₀² x² dx, N=4, h=0.5: (0.5/2)[0 + 2(0.25+1+2.25) + 4] = 2.75 (exact ≈2.667).

💡 REMEMBER

O(h²) error — halving h quarters the error. Works for ANY N, no parity constraint (unlike Simpson's).

METHOD 15
Simpson's 1/3 Rule

Use when: N is EVEN and you want much higher accuracy than trapezoidal at the same step size.

1Confirm N is even (required — parabolas span pairs of subintervals).
2Compute h = (b−a)/N.
3List nodes and f-values.
4Apply (h/3)[f₀ + 4f₁ + 2f₂ + 4f₃ + … + 4fₙ₋₁ + fₙ] — pattern 1,4,2,4,2,…,4,1.
MINI EXAMPLE

∫₀¹ eˣ dx, N=4, h=0.25: (0.25/3)[1+4(1.284)+2(1.6487)+4(2.117)+2.7183] ≈ 1.7183 (exact e−1≈1.71828).

💡 REMEMBER

O(h⁴) error — halving h shrinks error by 16× (vs 4× for trapezoidal). If N given is odd, you cannot use Simpson's directly.
